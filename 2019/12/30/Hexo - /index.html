⚑ 講義

撰寫 Blog 的意義：

  1. 想要鍛鍊自己

  2. 記錄百寶箱

    • 發現不會的可以翻舊的 Code
    • 許多同學的問題，其實我是給部落格寫過的內容

  4. 讓企業容易發覺你的價值（作品難分辨由誰執行，但部落格不好經營所以容易被信任）

  5. 放廣告加減賺（Ｘ，這其實很少）

Hexo 介紹：

  • 技術轉移

    • 純 AJAX

    • Jekyll:

      • ruby sass（資料量多，太慢10-100x差距）
      • 不須預先編譯

    • Hexo:

      • node.js sass

  • 繁中文件

    • 主要為台灣開發者
    • 眾多的中國協作者

  • 是一個靜態網站產生器 → 沒有資料庫（無法讓用戶改變內容）

  • 可以部署在 Github 上

    • 免費
    • 有著 Github 超潮網域

  • 使用 Markdown 撰寫

    • 專注在寫文之上
    • 缺點是沒辦法主動帶圖片 （可解決:上傳圖庫 ex firebase 46:14）

  • 大量地 Theme

    • 也可以自己做 Theme

  • 額外插件 plugins

    • https://hexo.io/plugins/
    • 搜尋、gzip、sitemap、rss

  • 外部工具

    • 留言版、Facebook 社群工具、GA …

  • 資料夾

    • package.json vs node_modules

    • copy

      • scaffolds（範例）/post.md

      • source（內容）/_posts/xxx.md

        • 變數 vs file name

          • title
          • date
          • tags

      • db.json(file name link )

    • themes（主題、佈景）

    • .gitignore

      • wcc723/wcc723.github.io（儲存庫）。other掛在wcc723.github.io之下。

      • branch x2 :

        • develop
        • master - 產生的靜態網頁、日期覆蓋

      • git嵌入，佈置到github（local to online）

firebase：

  • 圖庫金流參考 https://firebase.google.com/pricing/

    • 3種方案：

      • spark plan
      • flame plan

    • 計算（blaze plan）：

      • 每日流量約 800 ~ 1200
      • 實際週期： 11/1 ~ 11/14 早上

      • storage

        • GB stored

        • 下載請求數（download operation）：13 * 0.004 = 0.052
